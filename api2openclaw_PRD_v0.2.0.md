## 1. 产品概述

**api2openclaw** 是一个本地大模型统一接入与格式转换中间件，配套提供 Web 管理控制台。

解决三个核心问题：

1. 本地部署的开源模型（Qwen、LLaMA、Mistral 等）输出标准 JSON，而下游消费方要求 OpenClaw 兼容的纯文本格式，两者之间缺乏适配层。
2. 多个本地模型实例需要对外统一提供访问入口，缺乏标准化的 API Key 管理与 BaseURL 分发机制。
3. 管理员需要一个带登录保护的可视化界面，完成模型接入配置、Key 分发、用量查看等日常运维操作。

---

## 2. 背景与问题

### 2.1 现状

| 问题 | 描述 |
|------|------|
| 格式不兼容 | 开源模型（Ollama / vLLM 部署）返回 OpenAI JSON，但 OpenClaw 客户端期望特定纯文本结构 |
| 缺乏统一入口 | 多模型实例散布在不同端口/机器，用户需手动维护地址 |
| 无鉴权机制 | 本地模型服务无认证，任何内网用户均可直接调用 |
| 无可观测性 | 无法追踪各用户/团队调用量、错误率和延迟 |
| 无管理界面 | 所有配置只能通过配置文件或命令行操作，运维门槛高 |

### 2.2 期望状态

- 任意开源模型的输出，经 api2openclaw 转换后，可被 OpenClaw 客户端直接消费
- 管理员通过带登录保护的 Web 控制台，完成模型接入、Key 管理、用量查看
- 下游用户只需知道一个 BaseURL + 一个 API Key，即可接入任意后端模型

## 4. 产品目标与成功指标

### 4.1 目标

- G1：消除格式转换的手工成本，开发者零改动接入 OpenClaw
- G2：提供统一的 API Key 分发体系，支持多租户隔离
- G3：提供带登录保护的可视化管理控制台，降低运维门槛
- G4：提供足够的可观测性，让管理员可以做资源管控

### 4.2 成功指标

| 指标 | 目标值 |
|------|--------|
| 格式转换成功率 | ≥ 99.5% |
| 接口平均延迟增量（中间件自身引入） | ≤ 20ms P99 |
| API Key 创建到生效时间 | ≤ 5 秒 |
| 首次部署耗时（参考文档操作） | ≤ 30 分钟 |
| 控制台登录响应时间 | ≤ 500ms |
| 模型配置保存到路由生效时间 | ≤ 3 秒（无需重启服务） |

**技术栈建议（供研发参考，可调整）**

| 层 | 建议技术 | 说明 |
|----|----------|------|
| Web UI 前端 | React + Vite | 轻量，易于打包为静态资源由服务端托管 |

| 配置热重载 | 监听数据库变更，无需重启进程 | 模型配置修改 ≤ 3 秒生效 |

## 6. 核心功能模块

### 6.1 管理控制台（Web UI）

**功能描述**

独立的 Web 应用，打包后由 api2openclaw 服务静态托管（访问根路径 `/`），管理员通过浏览器访问，完成所有运维操作。

**页面路由结构**

```
/                              登录页
/dashboard                     登录后默认落地页（重定向至 /dashboard/overview）
/dashboard/overview            仪表盘概览
/dashboard/models              模型配置管理
/dashboard/keys                API Key 管理
/dashboard/logs                调用日志
```

**详细需求**

- [ ] 未登录状态下访问任意 `/dashboard/*` 路由，前端路由守卫自动跳转至 `/`
- [ ] 登录成功后跳转至 `/dashboard/overview`
- [ ] 退出登录后清除会话，跳转至 `/`
- [ ] 侧边栏导航高亮当前所在页面
- [ ] 顶部状态栏实时显示服务健康状态（绿色"运行中" / 红色"异常"，每 30 秒轮询 `/admin/health`）
- [ ] 界面语言为中文，响应式布局，最低兼容 1280px 宽度桌面浏览器

**验收标准**

- 直接访问 `/dashboard/models`，未登录时重定向至 `/`，登录后正常展示
- 刷新页面后，会话有效期内已登录状态保持，无需重新登录

---

### 6.2 登录与身份认证

**功能描述**

管理控制台的登录入口，支持用户名/密码认证。v0.2 仅支持单管理员账号，账号凭证通过环境变量初始化。

**详细需求**

- [ ] 登录页面包含：Logo 区域、用户名输入框、密码输入框（内容掩码）、登录按钮
- [ ] 点击登录按钮或在密码框按回车，均可触发登录请求
- [ ] 登录请求期间：按钮文字变为"登录中…"并禁用，防止重复提交
- [ ] 登录成功：服务端颁发 JWT，写入 HttpOnly Cookie，有效期 8 小时，跳转至 `/dashboard/overview`
- [ ] 登录失败：页面内联展示错误提示"用户名或密码错误"，不区分具体字段（防账号枚举攻击），输入框内容保留
- [ ] 连续登录失败 5 次（同一 IP），锁定该 IP 15 分钟，后续请求立即返回提示"登录尝试过多，请 15 分钟后再试"
- [ ] 退出登录：调用 `POST /admin/auth/logout`，服务端将 JWT 加入黑名单，清除 Cookie，跳转 `/`
- [ ] 密码存储：bcrypt hash，cost factor ≥ 10，不可逆存储
- [ ] v0.2 不支持在线找回密码，忘记密码需更新 `ADMIN_PASSWORD_HASH` 环境变量并重启服务

**初始账号配置（环境变量）**

```bash
ADMIN_USERNAME=admin
ADMIN_PASSWORD_HASH=$2b$10$xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
# bcrypt hash 可通过 CLI 工具生成：
# npx bcrypt-cli "your_password"
```

**Cookie 安全属性要求**

| 属性 | 值 | 说明 |
|------|----|------|
| HttpOnly | true | 禁止 JS 读取，防 XSS |
| SameSite | Strict | 防 CSRF |
| Secure | true（生产环境） | 仅 HTTPS 传输 |
| Path | /admin | 限制 Cookie 作用域 |

**验收标准**

- 正确账号登录 → 跳转 `/dashboard/overview`，Response Set-Cookie 包含有效 JWT
- 错误密码登录 → 停留登录页，展示错误提示，不返回具体失败原因
- 连续失败 5 次后第 6 次 → 立即返回锁定提示，后端不执行密码验证逻辑
- 退出后访问 `/dashboard/*` → 重定向至 `/`，旧 JWT 无效

### 6.3 模型配置管理

**功能描述**

管理员通过此模块注册本地模型实例，配置其 BaseURL 和 API Key，查看在线状态。配置变更 ≤ 3 秒内热重载至路由器，无需重启服务。

**页面：模型列表（`/dashboard/models`）**

每条模型记录展示以下字段：

| 字段 | 展示说明 |
|------|----------|
| 对外别名 | 下游用户请求时使用的模型名，如 `gpt-4` |
| 实际模型标识 | 后端模型真实名称，如 `qwen2.5-72b-instruct` |
| BaseURL | 模型服务地址 |
| API Key | 脱敏展示，仅显示前 6 位 + `••••••••`，不提供查看明文功能 |
| 在线状态 | 实时健康检查：在线（绿点）/ 离线（红点）/ 检查中（灰点） |
| 操作 | 编辑 / 删除 |

**详细需求——列表**

- [ ] 页面加载时调用 `GET /admin/models` 获取列表
- [ ] 每个模型在线状态每 30 秒自动刷新（轮询 `/admin/health`）
- [ ] 点击"删除"弹出二次确认弹窗（文案见第 10 节）
- [ ] 若被删除模型有 API Key 绑定，弹窗内额外展示绑定 Key 数量警告

**详细需求——添加/编辑表单（Inline 展开，非弹窗）**

表单字段：

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|----------|
| 对外别名 | 文本输入 | 是 | 1–64 字符；仅允许字母、数字、`-`、`_`、`.`；全局唯一 |
| 实际模型标识 | 文本输入 | 是 | 1–128 字符，与后端 `model` 字段一致 |
| BaseURL | URL 输入 | 是 | 合法 HTTP/HTTPS URL；格式示例 `http://host:port/v1`；末尾不含 `/` |
| API Key | 文本输入 | 否 | 若后端无需鉴权可留空；有值时长度 ≥ 8 |
| 备注 | 文本输入 | 否 | 最多 200 字符 |

表单行为：

- [ ] 保存前进行前端字段校验，不通过时字段边框变红，展示具体错误文字，不发起请求
- [ ] BaseURL 输入框失焦时，自动调用 `POST /admin/models/test` 检测连通性，展示检测结果
- [ ] 别名重复时，保存请求返回 409，前端在别名字段下展示"此别名已存在"
- [ ] 保存成功后：列表自动刷新，新增记录出现在末尾，短暂高亮 1.5 秒；右上角 Toast 提示"模型配置已保存并生效"
- [ ] 编辑模式：表单回填现有数据，API Key 字段显示占位符 `••••••••`，用户主动修改才更新，否则服务端保留原加密值不变
- [ ] 保存操作原子性：数据库写入成功后，路由器通过事件通知立即热重载（不轮询）

**验收标准**

- BaseURL 填写并失焦后，3 秒内展示连通性检测结果
- 别名与已有模型重复时，保存后字段下展示错误，不关闭表单
- 新增模型保存后，不重启服务，立即可通过该别名调用推理接口
- 删除模型后，路由至该模型的请求返回 `404`，错误码 `model_not_found`

### 6.4 API Key 管理

**功能描述**

管理员创建、查看、吊销供下游用户使用的 API Key。每个 Key 可绑定到指定模型别名，或允许调用全部模型。

**页面：Key 列表（`/dashboard/keys`）**

| 字段 | 展示说明 |
|------|----------|
| 标签 | 便于识别的名称，如"团队A"、"开发者-张三" |
| Key 值 | 脱敏展示（`sk-a2oc-XXXXXX...XXXX`） |
| 绑定模型 | 可调用的模型别名，"全部"表示无限制 |
| 有效期 | 永久 / 到期日期 |
| 创建时间 | YYYY-MM-DD |
| 状态 | 活跃（绿）/ 已吊销（红）/ 已过期（灰） |
| 操作 | 吊销（活跃状态可用）/ 删除记录（已吊销/过期可用） |

**详细需求**

- [ ] Key 格式：`sk-a2oc-{32 位随机字母数字}`，由服务端生成
- [ ] 创建成功后，Key 明文仅在弹窗中展示一次，关闭后不可再查，弹窗提供一键复制按钮
- [ ] 数据库仅存储 Key 的 bcrypt hash 与前缀（用于脱敏展示），明文不落库
- [ ] 支持设置有效期：不填则永久有效；可指定具体日期
- [ ] 吊销操作：弹出确认弹窗，确认后 ≤ 1 秒内所有节点生效
- [ ] 已吊销/过期的 Key 保留在列表中，可手动删除记录
- [ ] 列表支持按状态筛选（全部 / 活跃 / 已吊销 / 已过期）

**创建 Key 表单字段**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 标签 | 文本 | 是 | 1–64 字符，用于识别 Key 用途 |
| 绑定模型 | 下拉单选 | 是 | 从已注册模型别名列表中选择，或选"全部" |
| 有效期 | 日期选择器 | 否 | 不填则永久有效 |
| 备注 | 文本 | 否 | 最多 200 字符 |

**验收标准**

- 创建成功弹窗展示完整 Key，一键复制后按钮文字变为"已复制 ✓"，3 秒后复原
- 关闭弹窗后，`GET /admin/keys/:id` 不返回明文 Key
- 使用已吊销 Key 调用推理接口 → `401`，错误码 `key_revoked`
- 使用已过期 Key 调用推理接口 → `401`，错误码 `key_expired`
- 使用未绑定权限的模型别名调用 → `403`，错误码 `model_not_allowed`

### 6.5 格式转换引擎（Format Converter）

**功能描述**

将本地模型返回的标准 JSON（OpenAI 兼容格式）转换为 OpenClaw 期望的纯文本输出格式，支持流式（streaming）和非流式两种模式。

**详细需求**

- [ ] 解析 `choices[0].message.content`，输出为 OpenClaw 纯文本格式
- [ ] 支持 `stream: true`，将 SSE chunks 转换为 OpenClaw 流式协议
- [ ] 支持 tool call 格式标准化（将 JSON function call 转换为 OpenClaw 工具调用文本语法）
- [ ] 支持 role 映射：`system / user / assistant` → OpenClaw 对应角色标记
- [ ] 转换规则支持插件化扩展，方便适配不同版本的 OpenClaw 协议
- [ ] 转换失败时保留原始响应，响应头附加 `X-Conversion-Error: <错误描述>`

**验收标准**

- 对比标准测试集，纯文本输出与预期完全一致
- streaming 模式下首 token 延迟 ≤ 50ms（扣除模型推理延迟）
- tool call 转换后可被 OpenClaw 正确解析并执行

---

### 6.6 模型路由器（Model Router）

**功能描述**

统一管理多个本地模型实例，对外暴露单一接入点，按规则将请求路由至合适的模型后端。配置来源为数据库，由控制台维护，变更实时生效。

**详细需求**

- [ ] 支持注册多个模型实例，配置来源为数据库（不依赖静态配置文件）
- [ ] 支持模型别名映射（请求 `gpt-4` 自动路由至 `qwen2.5-72b-instruct`）
- [ ] 支持轮询（Round Robin）和加权轮询两种负载均衡策略
- [ ] 后端实例不可用时自动摘除，恢复后自动重新加入（健康检查间隔默认 30 秒，可配）
- [ ] 支持按 API Key 绑定固定模型，实现路由隔离
- [ ] 控制台保存配置后，路由器 ≤ 3 秒内通过事件通知完成热重载，无需重启

**验收标准**

- 某实例下线，路由器在 2 个健康检查周期内切换，请求无中断
- 别名映射后，响应体 `model` 字段返回实际模型名称（非别名）
- 并发 100 请求，轮询分布误差 ≤ 5%

### 6.7 用量监控与日志

**功能描述**

按 API Key 维度记录调用数据，支持配额限制、日志查询，数据在控制台日志页面展示。

**详细需求**

- [ ] 统计维度：API Key、模型名称、时间段（小时/天/月）
- [ ] 统计指标：请求次数、Token 消耗（prompt + completion）、错误率、P50/P99 延迟
- [ ] 控制台日志页支持按时间范围、API Key、模型、状态码筛选，分页展示
- [ ] 支持按 API Key 设置 RPM（每分钟请求数）和 TPD（每日 Token 上限）
- [ ] 超配额请求返回 `429`，响应头附加 `Retry-After`
- [ ] 日志保留 ≥ 90 天
- [ ] 支持导出 CSV 格式用量报告
- [ ] 告警规则：错误率 > 5% 或 P99 延迟 > 2s，触发 Webhook 通知（Webhook URL 在控制台配置）

**验收标准**

- 发起 1000 次请求后，监控统计数据误差 ≤ 0.1%
- 超限请求返回 429，`Retry-After` 秒后恢复正常
- CSV 导出文件字段与文档定义一致，数据与筛选条件匹配

## 8. 接口设计

### 8.1 推理接口（对外，OpenAI 兼容）

```
POST /v1/chat/completions
Authorization: Bearer <api_key>
Content-Type: application/json
```

请求体：

```json
{
  "model": "gpt-4",
  "messages": [{ "role": "user", "content": "你好" }],
  "stream": false
}
```

成功响应（200）：

```json
{
  "id": "chatcmpl-xxx",
  "object": "chat.completion",
  "model": "qwen2.5-72b-instruct",
  "choices": [{
    "message": {
      "role": "assistant",
      "content": "<openclaw:response>你好，有什么可以帮你？</openclaw:response>"
    },
    "finish_reason": "stop",
    "index": 0
  }],
  "usage": {
    "prompt_tokens": 10,
    "completion_tokens": 12,
    "total_tokens": 22
  }
}
```

错误响应（统一格式）：

```json
{ "error": { "code": "key_revoked",        "message": "API key has been revoked" } }
{ "error": { "code": "key_expired",         "message": "API key has expired" } }
{ "error": { "code": "model_not_found",     "message": "Model 'gpt-4' is not registered" } }
{ "error": { "code": "model_not_allowed",   "message": "This key is not allowed to access model 'gpt-4'" } }
{ "error": { "code": "rate_limit_exceeded", "message": "RPM limit reached" } }
{ "error": { "code": "quota_exceeded",      "message": "Daily token quota exceeded" } }
```

### 8.2 管理接口（需登录态 JWT Cookie）

**认证**

| 方法 | 路径 | 描述 |
|------|------|------|
| `POST` | `/admin/auth/login` | 用户名密码登录，颁发 JWT Cookie |
| `POST` | `/admin/auth/logout` | 退出登录，使 JWT 失效，清除 Cookie |
| `GET` | `/admin/auth/me` | 获取当前登录用户信息 |

`POST /admin/auth/login` 请求/响应：

```json
// 请求
{ "username": "admin", "password": "your_password" }

// 成功 200：设置 Set-Cookie，响应体
{ "username": "admin", "last_login_at": "2026-03-14T10:00:00Z" }

// 失败 401
{ "error": { "code": "invalid_credentials", "message": "用户名或密码错误" } }

// IP 锁定 429
{ "error": { "code": "too_many_attempts", "message": "登录尝试过多，请 15 分钟后再试", "retry_after": 900 } }
```

**模型管理**

| 方法 | 路径 | 描述 |
|------|------|------|
| `GET` | `/admin/models` | 获取模型列表（含在线状态） |
| `POST` | `/admin/models` | 创建新模型配置 |
| `PUT` | `/admin/models/:id` | 更新模型配置（触发热重载） |
| `DELETE` | `/admin/models/:id` | 删除模型配置 |
| `POST` | `/admin/models/test` | 测试 BaseURL 连通性（不写库） |

`POST /admin/models` 请求体：

```json
{
  "alias": "gpt-4",
  "model_id": "qwen2.5-72b-instruct",
  "base_url": "http://192.168.1.100:11434/v1",
  "api_key": "sk-local-xxxx",
  "note": "主力模型"
}
```

`POST /admin/models/test` 请求体与响应：

```json
// 请求
{ "base_url": "http://192.168.1.100:11434/v1", "api_key": "sk-local-xxxx" }

// 成功
{ "ok": true, "latency_ms": 42 }

// 失败（HTTP 仍 200，业务层失败）
{ "ok": false, "error": "connection refused" }
```

**API Key 管理**

| 方法 | 路径 | 描述 |
|------|------|------|
| `GET` | `/admin/keys` | 获取 Key 列表（脱敏） |
| `POST` | `/admin/keys` | 创建新 Key（响应体含一次性明文） |
| `DELETE` | `/admin/keys/:id` | 吊销 Key（status → revoked） |
| `GET` | `/admin/keys/:id/usage` | 获取指定 Key 的用量统计 |

`POST /admin/keys` 响应体（唯一一次返回明文）：

```json
{
  "id": "key_abc123",
  "label": "开发者-张三",
  "key": "sk-a2oc-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
  "model_alias": "gpt-4",
  "expires_at": null,
  "created_at": "2026-03-15T10:00:00Z"
}
```

**用量与日志**

| 方法 | 路径 | 描述 |
|------|------|------|
| `GET` | `/admin/usage` | 汇总用量统计（支持 `?key_id=&model=&from=&to=` 筛选） |
| `GET` | `/admin/logs` | 调用日志列表（分页 `?page=&limit=`，多维度筛选） |
| `GET` | `/admin/logs/export` | 导出 CSV（同筛选参数） |

**系统**

| 方法 | 路径 | 描述 |
|------|------|------|
| `GET` | `/admin/health` | 服务健康状态（含各模型实例在线状态） |
| `GET` | `/metrics` | Prometheus 格式指标 |

## 11. 非功能性需求

| 类别 | 需求 |
|------|------|
| 性能 | 中间件 P99 延迟增量 ≤ 20ms；支持 ≥ 200 QPS 并发 |
| 可用性 | 月度可用性 ≥ 99.9%；支持优雅重启，不丢失进行中请求 |
| 安全性 | API Key 以 bcrypt hash 验证；模型 API Key 以 AES-256 加密落库；管理接口全部需要登录态；生产环境强制 HTTPS；Cookie 设置 HttpOnly + SameSite=Strict + Secure |
| 可扩展性 | 格式转换规则插件化，新增 OpenClaw 协议版本无需改核心代码 |
| 可观测性 | `/metrics` 端点兼容 Prometheus；请求日志结构化输出 JSON 格式 |
| 部署 | 提供 Docker 镜像与 Docker Compose 一键部署；模型配置变更 ≤ 3 秒热重载，无需重启 |
| 兼容性 | 推理接口 100% 兼容 OpenAI `/v1/chat/completions` 规范 |

## 12. 依赖与约束

**技术依赖**

- 后端模型服务需暴露 OpenAI 兼容 HTTP 接口（Ollama / vLLM / LM Studio 均满足）
- 宿主机需支持 Docker 运行环境

**已知约束**

- v0.2 登录系统仅支持单管理员账号，不支持多账号/角色管理
- 当前版本不支持多模态（图片/音频）输入，仅处理文本
- OpenClaw 协议版本需在配置中显式指定，不自动探测
- 管理控制台不支持移动端，最低分辨率要求 1280px
- 忘记密码需更新环境变量并重启服务，v0.2 不提供在线找回功能
