## 1. 产品概述

**api2openclaw** 是一个本地大模型统一接入与格式转换中间件。

它解决两个核心问题：

1. 本地部署的开源模型（Qwen、LLaMA、Mistral 等）输出标准 JSON，而下游消费方要求 OpenClaw 兼容的纯文本格式——两者之间缺乏适配层。
2. 多个本地模型实例需要对外统一提供访问入口，但没有标准化的 API Key 管理与 BaseURL 分发机制。

api2openclaw 作为中间件，坐落在本地模型与下游用户/系统之间，提供格式转换、认证鉴权、路由调度、用量监控四大能力。

## 2. 背景与问题

### 2.1 现状

| 问题 | 描述 |
|------|------|
| 格式不兼容 | 开源模型（如通过 Ollama、vLLM 部署）返回标准 OpenAI JSON 格式，但 OpenClaw 客户端期望特定的纯文本输出结构 |
| 缺乏统一入口 | 多个模型实例散布在不同端口/机器，用户需手动维护地址，管理混乱 |
| 无鉴权机制 | 本地模型服务通常无认证，任何内网用户均可直接调用，存在安全与配额风险 |
| 无可观测性 | 无法追踪各用户/团队的调用量、错误率和延迟，难以做容量规划 |

### 2.2 期望状态

- 任意开源模型的输出，经 api2openclaw 转换后，可被 OpenClaw 客户端直接消费
- 管理员通过控制台统一管理 API Key 和模型路由规则
- 下游用户只需知道一个 BaseURL + 一个 API Key，即可接入任意后端模型

## 4. 产品目标与成功指标

### 4.1 目标

- **G1**：消除格式转换的手工成本，开发者零改动接入 OpenClaw
- **G2**：提供统一的 API Key 分发体系，支持多租户隔离
- **G3**：提供足够的可观测性，让管理员可以做资源管控

### 4.2 成功指标（OKRs）

| 指标 | 目标值 |
|------|--------|
| 格式转换成功率 | ≥ 99.5% |
| 接口平均延迟增量（中间件自身引入） | ≤ 20ms P99 |
| API Key 创建到生效时间 | ≤ 5 秒 |
| 首次部署耗时（参考文档操作） | ≤ 30 分钟 |

## 5. 核心功能模块

### 5.1 认证网关（Auth Gateway）

**功能描述**

为每位接入用户/团队颁发独立的 API Key，与 BaseURL 绑定，管理员可在控制台进行增删改查与配额设定。

**详细需求**

- [ ] 支持通过控制台或 REST API 创建 API Key
- [ ] API Key 支持设置有效期（永久 / 固定日期 / 相对天数）
- [ ] 每个 API Key 可绑定到指定模型或模型组，禁止跨权限调用
- [ ] 支持即时吊销（吊销后 ≤ 1 秒生效）
- [ ] 请求头格式兼容 OpenAI 标准：`Authorization: Bearer <api_key>`
- [ ] Key 泄露检测：连续异常大流量请求触发告警

**验收标准**

- 使用有效 Key 发起请求，返回 200
- 使用无效/过期 Key，返回 401，错误体包含 `error.code` 和 `error.message`
- 吊销 Key 后，下一个请求立即返回 401

---

### 5.2 格式转换引擎（Format Converter）

**功能描述**

将本地模型返回的标准 JSON（OpenAI 兼容格式）转换为 OpenClaw 期望的纯文本输出格式，支持流式（streaming）和非流式两种模式。

**详细需求**

- [ ] 解析 `choices[0].message.content` 并输出为 OpenClaw 纯文本格式
- [ ] 支持 `stream: true` 时，将 SSE chunks 转换为 OpenClaw 流式协议
- [ ] 支持 tool call 格式标准化（将 JSON function call 转换为 OpenClaw 工具调用文本语法）
- [ ] 支持 role 映射：`system / user / assistant` → OpenClaw 对应角色标记
- [ ] 转换规则支持插件化扩展，方便适配不同版本的 OpenClaw 协议
- [ ] 转换失败时，保留原始响应并在响应头附加 `X-Conversion-Error` 字段

**验收标准**

- 对比标准测试集，纯文本输出与预期完全一致
- streaming 模式下首 token 延迟不超过 50ms（扣除模型本身延迟）
- tool call 转换后可被 OpenClaw 正确解析并执行

---

### 5.3 模型路由器（Model Router）

**功能描述**

统一管理多个本地模型实例，对外暴露单一接入点，按规则将请求路由至合适的模型后端。

**详细需求**

- [ ] 支持注册多个模型实例（名称 + 地址 + 端口）
- [ ] 支持模型别名映射（例：请求 `gpt-4` 自动路由至 `Qwen-72B`）
- [ ] 支持轮询（Round Robin）和加权轮询两种负载均衡策略
- [ ] 后端实例不可用时自动摘除，恢复后自动重新加入（健康检查间隔可配）
- [ ] 支持按 API Key 绑定固定模型，实现路由隔离
- [ ] 管理员可通过控制台实时修改路由规则，无需重启服务

**验收标准**

- 某实例下线，路由器在 2 个健康检查周期内完成切换，请求无中断
- 别名映射后，响应体中 `model` 字段返回实际模型名称（而非别名）
- 并发 100 请求，路由分布误差 ≤ 5%（轮询策略下）

---

### 5.4 用量监控（Usage Monitor）

**功能描述**

按 API Key 维度记录调用数据，支持配额限制、日志导出和告警规则配置。

**详细需求**

- [ ] 统计维度：API Key、模型名称、时间段（小时/天/月）
- [ ] 统计指标：请求次数、Token 消耗（prompt + completion）、错误率、P50/P99 延迟
- [ ] 支持按 API Key 设置请求数限流（RPM）和 Token 限额（TPD）
- [ ] 超配额请求返回 429，响应头附加 `Retry-After`
- [ ] 操作日志保留 ≥ 90 天，支持按时间/Key/模型过滤查询
- [ ] 支持导出 CSV 格式用量报告
- [ ] 告警规则：错误率 > 5% 或 P99 延迟 > 2s，触发 Webhook 通知

**验收标准**

- 发起 1000 次请求后，监控数据误差 ≤ 0.1%
- 超限请求返回 429，并在 `Retry-After` 秒数后恢复正常
- 日志导出 CSV，字段与文档定义一致

## 6. 用户故事

```
作为 管理员，
我希望 能在控制台一键创建 API Key 并绑定指定模型，
以便 快速为新成员分配访问权限，而无需修改服务配置。

作为 开发者，
我希望 只需修改 BaseURL 和 API Key 就能将现有 OpenClaw 应用对接本地模型，
以便 以最低成本完成迁移，不改动任何业务代码。

作为 团队负责人，
我希望 能查看每个 API Key 的月度 Token 消耗报告，
以便 合理分摊 GPU 资源成本并做预算规划。

作为 开发者，
我希望 当我调用工具（tool call）时，输出格式被正确转换为 OpenClaw 语法，
以便 工具调用链路端到端可用，不需要我在业务层做额外解析。
```

## 7. 接口设计

### 7.1 推理接口（对外，兼容 OpenAI）

```
POST /v1/chat/completions
Authorization: Bearer <api_key>
Content-Type: application/json
```

**请求体示例**

```json
{
  "model": "gpt-4",
  "messages": [
    { "role": "user", "content": "你好" }
  ],
  "stream": false
}
```

**响应体**（转换后，OpenClaw 纯文本格式，封装于 content 字段）

```json
{
  "id": "chatcmpl-xxx",
  "object": "chat.completion",
  "model": "Qwen-72B",
  "choices": [{
    "message": {
      "role": "assistant",
      "content": "<openclaw:response>你好，有什么可以帮你？</openclaw:response>"
    }
  }],
  "usage": {
    "prompt_tokens": 10,
    "completion_tokens": 12,
    "total_tokens": 22
  }
}
```

### 7.2 管理接口（内部，需 Admin Token）

| 方法 | 路径 | 描述 |
|------|------|------|
| `GET` | `/admin/keys` | 列出所有 API Key |
| `POST` | `/admin/keys` | 创建新 API Key |
| `DELETE` | `/admin/keys/:id` | 吊销 API Key |
| `GET` | `/admin/models` | 列出已注册模型实例 |
| `POST` | `/admin/models` | 注册新模型实例 |
| `GET` | `/admin/usage` | 查询用量统计 |
| `GET` | `/admin/logs` | 查询操作日志 |

## 8. 非功能性需求

| 类别 | 需求 |
|------|------|
| **性能** | 中间件自身引入的 P99 延迟 ≤ 20ms；支持 ≥ 200 QPS 并发 |
| **可用性** | 服务可用性 ≥ 99.9%（月度）；支持优雅重启，不丢失进行中请求 |
| **安全性** | API Key 存储加密；管理接口仅限内网访问；支持 HTTPS |
| **可扩展性** | 格式转换规则插件化，新增 OpenClaw 版本适配不需改核心代码 |
| **可观测性** | 暴露 `/metrics` 端点兼容 Prometheus；结构化日志输出（JSON 格式） |
| **部署** | 提供 Docker 镜像和 Docker Compose 一键部署方案；支持配置文件热重载 |
| **兼容性** | 推理接口 100% 兼容 OpenAI `/v1/chat/completions` 规范 |

## 9. 依赖与约束

**技术依赖**

- 后端模型服务需暴露 OpenAI 兼容 HTTP 接口（Ollama / vLLM / LM Studio 均满足）
- 宿主机需支持 Docker 运行环境

**已知约束**

- 当前版本不支持多模态（图片/音频）输入，仅处理文本
- OpenClaw 协议版本需在配置中显式指定，不自动探测
- 管理控制台为 Web UI，移动端适配不在 v0.1 范围内
